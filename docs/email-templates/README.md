# Supabase Auth email templates

Hand-written, brand-consistent HTML templates for every Supabase Auth
email FormulaNorth sends. Pasted into the Supabase dashboard at
**Project Settings → Auth → Email Templates**.

## File map

| File | Supabase template | Subject line |
|---|---|---|
| `confirm-signup.html` | Confirm signup | `Confirm your FormulaNorth account` |
| `invite-user.html` | Invite user | `You've been invited to FormulaNorth` |
| `magic-link.html` | Magic link | `Sign in to FormulaNorth` |
| `change-email.html` | Change email address | `Confirm your new FormulaNorth email` |
| `reset-password.html` | Reset password | `Reset your FormulaNorth password` |
| `reauthentication.html` | Reauthentication | `Your FormulaNorth verification code` |

## Why these are version-controlled (not just pasted into Supabase)

Supabase doesn't track template diffs. If we edit a template in the
dashboard and break it, there's no rollback. Keeping the canonical
copy in the repo means:

- Edits are reviewable
- We can re-paste the known-good version if something breaks
- The brand language stays consistent across all 6 templates
- New team members can see how each email is structured

When you change a template:

1. Edit the file here in the repo
2. Commit
3. Open Supabase dashboard → paste the updated HTML in
4. Save

## Design constraints

These templates are hand-written for cross-client compatibility, not
generated by a builder like MJML or Foundation. Constraints:

- **Inline CSS only** — Gmail strips `<style>` blocks
- **Table-based layout** — Outlook desktop ignores `display:flex` / `grid`
- **System font stack** — no external fonts (Google Fonts breaks in many clients)
- **Absolute https URLs** — image and link URLs must be fully qualified
  (`https://formulanorth.ca/FormulaNorth_Logo.png`, not `/...`)
- **Width 600px max** — standard email-safe width, scales down on mobile
- **Preheader hidden div** — first ~100 chars show in inbox preview line

## Available Supabase template variables

Go-template syntax. Not all variables are available in all templates.

| Variable | Available in | Notes |
|---|---|---|
| `{{ .ConfirmationURL }}` | Confirm, Invite, Magic, Change, Reset | Action URL |
| `{{ .Token }}` | Magic, Reset, Reauthentication | 6-digit OTP |
| `{{ .TokenHash }}` | Same as Token | Hashed version |
| `{{ .Email }}` | All | Recipient email |
| `{{ .NewEmail }}` | Change email only | The new address |
| `{{ .SiteURL }}` | All | Site URL set in Supabase config |
| `{{ .Data }}` | All | User-defined metadata (custom) |

## Year footer

Hardcoded `&copy; 2026 FormulaNorth`. Bump it once a year — Supabase
templates have no `{{ now.Year }}` helper.

## Testing checklist when changing a template

After saving in Supabase:

- [ ] Open formulanorth.ca in incognito
- [ ] Trigger the email (sign up / forgot password / etc.)
- [ ] Open in Gmail web — logo and button render, no broken layout
- [ ] Open in Gmail mobile — same checks at narrow width
- [ ] Open in Apple Mail — same
- [ ] Click the action button — lands on the correct page
- [ ] Click each footer link — privacy / terms / contact all valid

If anything looks broken, the template was probably saved with stripped
HTML (Supabase's editor sometimes auto-trims). Paste again from the
file in this repo — it's the source of truth.
